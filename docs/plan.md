## PHASE 2: Playing Proto A をベースにした対戦ゲーム実装計画

### 概要

現在の Playing Proto A（縦画面、ピッチャー上半分・バッター下半分）をそのまま使い、
BSO カウント・イニング管理・スコアボードを追加して9回制の対戦ゲームとして完成させる。

### UI 仕様

#### ステータスバー（画面上端、壁ラインの上）

```
青軍 3  B●●○ S●○○ O○  1回表
赤軍 1
```

- 2行構成
- チーム名: 「青軍」「赤軍」（ひとまず固定）
- 得点: 数字
- B（ボール）: 最大3個、●で現在カウント
- S（ストライク）: 最大2個
- O（アウト）: 最大2個
- イニング: 「1回表」「1回裏」…「9回裏」
- フォント小さめ、1行に収める

---

### ゲームルール

#### 攻守

- 攻撃側: バッター操作（画面下半分でスワイプ）
- 守備側: ピッチャー操作（画面上半分でスワイプ）
- 表: 青軍攻撃・赤軍守備
- 裏: 赤軍攻撃・青軍守備

#### ストライク・ボール判定（現行実装をそのまま使用）

| 結果 | 条件 |
|------|------|
| STRIKE | バウンドなしでストライクゾーン通過 |
| BALL | バウンドあり・ゾーン外通過・画面端素通り |
| MISS（空振り） | バットが当たらなかった場合 → ストライク扱い |
| HIT | バットに当たった → 打球フェーズへ |

※ 現行実装ではMISS廃止済みのため、バット距離16px以内 = HIT。バット未接触でゾーン通過ならSTRIKE/BALL。

#### カウント進行

- ストライク（STRIKE / MISS / 空振り）: S +1
- ボール（BALL / バウンド）: B +1
- 3ストライク → 三振アウト
- 4ボール → フォアボール（走者1塁へ）

#### アウト

- 三振
- 走者が塁到達前にボールが塁に当たる（現行実装）
- 2アウトでチェンジ

#### 得点

- 走者がホームインで1点（現行実装の showPlayingScore() をスコアに反映）

#### イニング進行

- 表: 青軍攻撃（3アウトでチェンジ）
- 裏: 赤軍攻撃（3アウトでチェンジ）
- 9回裏終了 or 裏の途中で決着 → ゲームセット

#### 試合終了

- 9イニング終了後、得点の多い方が勝ち
- 同点の場合: 引き分け（延長なし・初版）

### 追加する処理フロー

#### 1球の流れ

```
ピッチャーが投球
  → ゾーン通過 → STRIKE/BALL 判定 → カウント更新
  → バット接触 → HIT → 走者フェーズ
    → 走者がホームイン → 得点加算
    → 全走者停止 → 打席終了 → 次打者へ
  → 3S → 三振 → アウト +1
  → 4B → フォアボール → 走者1塁
```

#### チェンジ

```
アウト3 → チェンジ演出（「チェンジ！」表示）
  → isTop 反転
  → outs=0, balls=0, strikes=0, runners=[] リセット
  → 9回裏終了チェック → ゲームセット or 継続
```

#### ゲームセット

```
「ゲームセット！」表示
「○○軍の勝ち！」or「引き分け！」
スコア最終表示
リスタートボタン
```

## 目的

GitHub Pages で公開できる、2人対戦の野球盤ゲームを作る。

- 2人対戦専用
- 1台のスマホでプレイ可能
- マルチタッチ対応で2人が同時操作可能
- ピッチャー操作とバッター操作を分離
- 投球から打撃まではマウンド付近の拡大画面
- 打球発生後は球場全体画面へ切り替え
- 試合履歴を IndexedDB に保存

## 実機調査から採用するコア要素

実機の野球盤では、近年モデルを中心に以下の特徴がある。

- 投手がコースを投げ分ける
- 高低を含む投球表現がある
- 打者側はタイミングとコース対応で打つ
- 打球結果は盤面上のゾーンで決まる
- 守備位置や守備シフトの概念がある
- BSO とイニング進行がある

今回のブラウザ版では、見た目の完全再現ではなく、駆け引きが成立する部分を優先して再現する。

## 画面構成

全画面はスマホ横持ちを基準に設計する。
PC でも動かすが、初版の最適体験はスマホ横持ち 1 画面 2 人対戦とする。

### 1. タイトル画面

- 対戦開始
- 履歴を見る
- ルール確認
- スマホ横持ち推奨の案内

### 2. 打席画面

マウンドからホームベース周辺を大きく見せる画面。

画面は左右にプレイヤー専用のタッチ領域を持つ。

- 左側: ピッチャー操作領域
- 右側: バッター操作領域
- 中央: 投球と打撃の演出表示領域

- 投手UI
  - 球種選択
  - 高低左右のコース選択
  - 投球確定
- 打者UI
  - 待機位置選択
  - スイング入力
  - 見逃し
- 共通表示
  - BSO
  - inning
  - 走者状況
  - 前球結果

### 3. 打球画面

球場全体を俯瞰で見せる画面。

- 打球方向
- 打球の強さ
- フェア / ファウル
- ゾーン結果
  - OUT
  - 1BH
  - 2BH
  - 3BH
  - HR
- 守備シフトの影響

### 4. 試合結果画面

- スコアボード
- 各打席ログ
- 保存完了表示
- 再戦

## モバイル前提のUI要件

- 片手親指操作を前提に大きいタップ領域を使う
- 左右のプレイヤー領域が誤タップで干渉しない余白を確保する
- 2点以上の同時タッチを Pointer Events で処理する
- 長押しやドラッグより、タップと短いスライド中心にする
- 打席画面は常時横向き前提レイアウトとする
- 画面サイズが不足する場合は簡易表示に切り替える

## マルチタッチ対戦の前提

1台のスマホを向かい合って持つ、または机に置いて左右から操作する形を基本とする。

- 投手は左半分の UI にだけ触れる
- 打者は右半分の UI にだけ触れる
- 同時タッチ時は pointerId 単位で入力を追跡する
- 相手側の操作領域への入力は無効化する
- 演出中も次操作に必要な UI だけは残して入力待ちを明確にする

## 操作モデル

## 投手

投手は 1 球ごとに以下を選ぶ。

- 球種
  - ストレート
  - スライダー
  - カーブ
  - フォーク
  - 消える魔球
- コース
  - 高 / 中 / 低
  - 内角 / 中央 / 外角

スマホ操作では、左親指で球種、右親指でコースを選ぶのではなく、
投手側 1 人が左側領域内で完結できるようにする。

- 左下: 球種ボタン群
- 左上: 3 x 3 コースグリッド
- 左中央: 投球確定ボタン

初版では 3 x 3 の 9 マスを採用する。
球種は見た目と判定補正に使い、物理演算の複雑化は避ける。

## 打者

打者は投球前または投球中に以下を操作する。

- 狙い位置
  - 高 / 中 / 低
  - 引っ張り / センター / 流し
- スイングタイミング
  - 早い
  - ちょうど
  - 遅い
- 見逃し

スマホ操作では、打者は右側領域だけで完結する。

- 右上: 高 / 中 / 低 の狙い位置
- 右下: 引っ張り / センター / 流し
- 右中央: スイングボタン
- 右端: 見逃しボタン

初版では、打者の入力負荷を下げるため、
リアルタイム反射神経ゲームではなく短い入力ウィンドウ方式にする。

## 1打席の進行

1. 投手が球種とコースを選ぶ
2. 打席画面で投球アニメーション開始
3. 打者が右側タッチ領域で狙いとスイングタイミングを入力
4. ミート判定を計算
5. 空振り / 見逃し / ファウル / インプレーを決定
6. インプレー時に球場全体画面へ切り替え
7. 打球ゾーンと守備シフトから結果を決定
8. 走者進塁、得点、アウト、BSO を更新
9. 打席ログを記録

## 判定の基本方針

### ミート判定

入力差分を数値化して結果を出す。

- pitchLocation と aimLocation の距離
- pitchHeight と swingHeight の一致度
- timing 差
- pitchType ごとの補正

結果カテゴリは以下。

- 見逃しストライク / ボール
- 空振り
- ファウル
- ゴロ
- ライナー
- フライ
- 長打
- ホームラン

### 打球結果

打球画面では実機盤面のポケット感を残すため、
最終結果をゾーンテーブルで扱う。

- レフト線
- 左中間
- センター
- 右中間
- ライト線
- 内野左
- 二遊間
- 一二塁間
- 内野右

各ゾーンに対して以下を割り当てる。

- OUT
- 1BH
- 2BH
- 3BH
- HR

守備シフトで OUT 率とヒット率を少し変える。

## 守備

プレイヤーが毎球細かく守備人形を動かす方式にはしない。
初版は守備シフト選択方式にする。

- 定位置
- 引っ張り警戒
- 流し打ち警戒
- 前進守備
- 長打警戒

これで実機の守備配置の駆け引きを軽量に再現する。

## 試合ルール初版

- 3イニングまたは 6イニングを選択可能
- 表裏あり
- 3アウトチェンジ
- 延長なし、同点なら引き分け
- DH や盗塁は初版では省略

## IndexedDB 保存方針

### 保存単位

1試合を 1 レコードとして保存する。

### 保存内容

- matchId
- playedAt
- innings
- homeTeamName
- awayTeamName
- finalScore
- winner
- turns

### turns に含める内容

- inning
- half
- batterSide
- pitchType
- pitchCourse
- batterAim
- swingTiming
- contactResult
- battedBallZone
- playResult
- outsAfter
- basesAfter
- scoreAfter

### 利用画面

- 履歴一覧
- 試合詳細
- 同条件で再戦

## 技術方針

- GitHub Pages 配置を前提に静的サイトで構成
- フロントエンドは TypeScript ベース
- ビルド後は静的ファイルを dist に出力
- IndexedDB は idb 系の軽量ラッパー利用を優先
- タッチ入力は Pointer Events を基準に実装する
- CSS は mobile-first で設計する
- 画面回転とセーフエリアを考慮する

## 推奨実装順

1. 静的ホスティング向けフロントエンド基盤作成
